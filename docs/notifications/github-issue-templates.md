# GitHub Issue Templates

## Autopilot sprint

Pouzi na planovanie konkretneho autonomneho sprintu.

Povinne polia: nazov, ciel, approval level, vystup, povolene subory, zakazane zmeny, checklist a dalsi sprint.

Labely: autopilot, sprint.

## QA bug report

Pouzi na reprodukovatelnu chybu alebo QA zistenie.

Povinne polia: severity, URL alebo subor, ocakavany stav, skutocny stav, kroky, dokaz, dopad, riziko, autofix, approval level, odporucana oprava.

Labely: bug, qa, pripadne critical, checkout, seo, ux alebo design.

## Owner decision needed

Pouzi, ked je potrebne rozhodnutie majitela pred pokracovanim.

Povinne polia: oblast, rozhodnutie, moznosti, odporucanie, dopad, riziko, co sa stane bez rozhodnutia a ci blokuje sprint.

Labely: owner-decision.

## Product open point

Pouzi na neovereny produktovy udaj.

Povinne polia: produkt alebo cluster, neovereny udaj, preco je dolezity, zdroj pravdy, odporucany stav, Status draft, Published 0 a co doplnit.

Labely: product, open-point.
