# Systém Produkt → Blog → Projekt → Predajná zostava

KomArena.sk nemá byť iba sklad súčiastok. Každý dôležitý produkt alebo produktová skupina má byť súčasťou systému:

Produkt → Blog/návod → Modelový projekt → Odporúčané produkty → Interné odkazy → Predajná zostava → Sociálne siete → Newsletter

## 1. Produktový popis

Ku každému dôležitému produktu priprav:

- SEO názov,
- short description,
- dlhý popis,
- technické parametre,
- kompatibilitu,
- obsah balenia,
- bezpečnostné odporúčania,
- záruku a vrátenie.

## 2. Profesionálny blog / návod

Navrhni minimálne jeden článok typu:

- Ako použiť [produkt] v ESP32 projekte
- Ako zapojiť [produkt] do smart home projektu
- Ako využiť [produkt] pri Home Assistant / ESPHome / ESPBuild riešení
- Najčastejšie chyby pri použití [produkt]
- Čo treba dokúpiť k [produkt]

Článok musí byť odborný, praktický, čitateľný a technicky opatrný. Nepíš neoverené zapojenia ako hotový fakt.

## 3. Modelový príklad zapojenia alebo použitia

Ku každému vhodnému produktu priprav modelový príklad:

- názov projektu,
- čo zákazník postaví,
- na čo to slúži,
- aké produkty z KomArena.sk potrebuje,
- aké doplnky odporúčame,
- základný princíp zapojenia,
- bezpečnostné upozornenia,
- možné rozšírenia,
- otvorené body, ak je potrebné overenie.

Príklady:

- ESP32 + teplotný senzor = meranie teploty do Home Assistant cez ESPHome
- ESP32 + relé modul = ovládanie nízkonapäťového zariadenia
- ESP32 + PIR senzor = detekcia pohybu pre smart home automatizáciu
- napájací modul + ESP32 = stabilné napájanie projektu
- Dupont vodiče + vývojová doska + senzor = základné prototypovanie

## 4. Odporúčané produkty z KomArena.sk

Pri každom produkte navrhni:

- hlavný produkt,
- potrebné príslušenstvo,
- odporúčané doplnky,
- alternatívy,
- vhodný kit alebo zostavu.

## 5. Predajný účel

Každý blog a projekt má pomáhať zákazníkovi:

- pochopiť problém,
- vybrať správny produkt,
- dokúpiť potrebné príslušenstvo,
- vyhnúť sa chybám,
- postaviť jednoduchý projekt,
- vrátiť sa na KomArena.sk po ďalšie komponenty.

## 6. Bezpečnostné pravidlo

Ak ide o napájanie, batérie, relé, vyššie napätie, prúd, teplo alebo zásah do elektroinštalácie:

- nepísať nebezpečné návody,
- jasne upozorniť, že práca s 230 V patrí odborne spôsobilej osobe,
- pri neistote použiť iba bezpečný nízkonapäťový modelový príklad,
- neuvádzať zapojenie ako finálne, ak nebolo overené.

## 7. Overiteľnosť

Každý technický článok musí rozlišovať:

- overené fakty,
- modelový príklad,
- odporúčanie,
- otvorený bod.

Pri neoverenej kompatibilite použi:

`Otvorený bod: kompatibilitu alebo zapojenie je potrebné overiť podľa konkrétneho modulu, datasheetu a konfigurácie.`

## 8. Interné prelinkovanie

Každý produkt má mať návrh:

- článku, ktorý ho vysvetľuje,
- projektu, kde sa použije,
- kategórie, do ktorej patrí,
- súvisiacich produktov,
- odporúčaného kitu alebo zostavy.

Každý článok má odkazovať na:

- hlavný produkt,
- potrebné príslušenstvo,
- kategóriu,
- ďalší nadväzujúci článok.

## 9. Povinný výstup pri každom budúcom produkte

Pri každom budúcom produkte má Codex pripraviť aj tieto polia:

- Product name
- Product short description
- Product long description
- Technical parameters
- Safety notes
- Suggested blog title
- Suggested blog outline
- Suggested model project
- Required KomArena products
- Recommended accessories
- Internal links
- Open points
